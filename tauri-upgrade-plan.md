# Tauri版本升级计划

## 当前状态
- **Tauri版本**: 1.5 (发布于2024年6月)
- **状态**: 较旧版本，可能存在兼容性问题

## 推荐升级目标
- **目标版本**: Tauri 2.0.0
- **发布日期**: 2024年10月
- **状态**: 稳定版本，长期支持

## 升级原因
1. **更好的兼容性** - 2.0支持更新的操作系统和工具链
2. **性能改进** - 更快的启动时间和资源使用
3. **新特性** - 更好的API和开发体验
4. **安全更新** - 修复了1.x版本的安全问题
5. **社区支持** - 大多数新项目和教程使用2.0

## 升级步骤

### 步骤1: 更新Cargo.toml
```toml
[dependencies]
tauri = { version = "2.0.0", features = [ 
  "path-all", 
  "dialog-open", 
  "dialog-save", 
  "fs-read-file", 
  "fs-write-file", 
  "shell-open" 
] }

[build-dependencies]
tauri-build = { version = "2.0.0", features = [] }
```

### 步骤2: 更新配置文件
Tauri 2.0的配置结构有一些变化：
1. 移除旧的配置属性
2. 使用新的安全配置
3. 更新窗口配置格式

### 步骤3: 更新代码
检查是否有不兼容的API变化：
1. `window.__TAURI__` 可能有变化
2. 事件系统更新
3. 插件系统改进

### 步骤4: 测试
1. 本地开发测试
2. 构建测试
3. 功能测试

## 风险评估

### 低风险
- 前端代码(Vue/JavaScript)基本不受影响
- 基本UI功能应该保持兼容

### 中风险
- Tauri API调用可能需要调整
- 配置文件格式变化

### 高风险
- 构建流程可能有变化
- 需要验证所有平台构建

## 时间估计
- **准备**: 1-2小时
- **实施**: 2-3小时  
- **测试**: 1-2小时
- **总计**: 4-7小时

## 备用方案
如果升级遇到问题，可以考虑：
1. **逐步升级**: 先到1.6，再到2.0
2. **保持1.5**: 如果功能满足需求且稳定
3. **创建新分支**: 在新分支试验升级

1. 备份当前项目

4. 测试每个步骤
